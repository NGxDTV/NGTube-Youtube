# Security Policy

## Supported Versions

NGTube is provided as an educational and research project.  
Only the **latest released version** is supported.

Security issues reported for outdated versions may not be addressed.

---

## Reporting a Vulnerability

If you discover a security-related issue, please report it **responsibly**.

### Please do NOT:
- Open a public GitHub issue for security vulnerabilities
- Share exploit details publicly before a fix is available

### How to Report

Send a report including:
- A clear description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Your environment (Python version, OS)

You may report security issues via:
- GitHub Issues **marked clearly as [SECURITY]**, or
- Direct contact with the project maintainer (if available)

---

## Scope

This security policy applies to:
- Code execution vulnerabilities
- Injection issues
- Unsafe deserialization
- Remote code execution
- Data exposure caused by the library itself

This project does **not** guarantee:
- Protection against YouTube rate limiting or blocking
- Stability of YouTube internal APIs
- Compliance with third-party Terms of Service

---

## Out of Scope

The following are **not considered security issues**:
- YouTube blocking or throttling requests
- Changes in YouTube internal APIs
- IP bans or captchas
- Legal or Terms of Service concerns
- Requests to bypass authentication, paywalls, or private content

Such reports will be closed without action.

---

## Responsible Disclosure

Please allow a reasonable amount of time for investigation and fixes before disclosing details publicly.

There is **no bug bounty program** associated with this project.

---

Thank you for helping keep NGTube secure.
